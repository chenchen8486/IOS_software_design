2.2.4 RF-03 初始化交互完整性

推导来源：RF-03 环境配置与飞机初始化  |  输入约束：2.1.3 数据依赖矩阵（AIRCRAFT / REF AIRPORT / CURRENT CONDITIONS 域）  |  输出产物：三层参数校验链、Save/Recall Scenario 持久化契约

RF-03 是 RF-01 主流程的前置准备阶段，负责训练开始前的飞机参数、参考机场、当前环境等初始化配置。其输出（初始化完成确认）是 RF-02 课程执行的前置条件。本节推导 RF-03 的交互序列、参数校验链以及 Save/Recall Scenario 的状态持久化契约。

前置阅读提示：本节序列图涉及的 SSM（Session State Manager）与 SC（Simulator Core）的详细架构定义参见 2.2.1 核心推导架构实体详解。

初始化交互序列图与参数校验链

RF-03 是纯逻辑推理流程，所有步骤都是精确的数值输入与包线验证。其交互序列的特点是多轮参数变更 + 层级校验。整体流程可分为三个串行阶段：

  • 飞机参数初始化：教员在 AIRCRAFT Tab 输入 GW、ZFW、Fuel 等重量参数，Control Board 先做 UI 层校验，SSM 再做业务层校验（等式、包线），通过后方可下发至 Simulator Core；
  • 机场与环境初始化：教员依次选择参考机场、跑道，并在 CURRENT CONDITIONS 中配置环境参数，SSM 依据机场标高与温度剖面完成 QNH/QFE 等派生参数的校验；
  • 全局目视确认：Map 渲染完整场景，教员确认通过后，SSM 将全局状态置为 INIT_COMPLETE，解锁后续 RF-02 课程执行流程。

阶段内部允许分支修正：任何一层校验失败均会阻断下发，并由 SSM 按对应策略回滚至上一次合法状态。

参数校验链的架构实现推导

RF-03 的初始化流程揭示了一个三层校验架构，这是 2.1.2 中"UI 层 → 状态管理层 → 模拟器核心层"数据流的具体实例：

三层校验层级定义

L1 UI 层校验
执行者：Control Board 前端控件
校验内容：输入格式（数字/字符串）、取值范围（如 GW 必须在 40t~80t 之间）
失败行为：即时阻止非法输入，给出红色边框/提示文案，不向 SSM 发送请求
L2 业务校验
执行者：Session State Manager
校验内容：跨参数等式（GW = ZFW + Fuel）、包线约束（CG 在 MAC 范围内）、依赖顺序（必须先选机场再选跑道）
失败行为：拒绝下发并回滚到上次合法状态，向 CB 返回具体错误码
L3 物理校验
执行者：Simulator Core
校验内容：物理可行性（如当前 GW、温度、标高、跑道长度条件下是否满足起飞性能越障要求）、地形碰撞检测、FMS 航线可计算性
失败行为：返回物理校验失败码，SSM 阻止本次参数生效并保持上一次合法配置，要求教员调整
失败回滚策略：L1 失败无需回滚（CB 前端拦截，非法请求不会到达 SSM）；L2 失败回滚到 SSM 上一次合法状态；L3 失败时 SC 返回错误码，SSM 阻止本次参数生效并保持上一次合法配置，要求教员调整。三层校验的粒度逐层收紧，避免了"前端通过但后端崩溃"的状态不一致。

Save/Recall Scenario 的状态持久化契约

ch1_8 的 4.3 节描述了 Advanced → Environment 中的 Save/Recall Scenario 功能。将其映射到 2.1 架构中，需要明确保存范围与数据依赖矩阵的对应关系。

Weather Scenario 数据项2.1.3 数据类别是否被 Save/Recall 持久化架构原因
海平面温度、中间层/对流层顶温度环境参数✅ 保存属于教员输入的配置参数，可复用
QNH（高度表设定）环境参数✅ 直接保存教员直接输入的高度表设定值，Recall 时原样恢复
QFE（场面气压）环境参数（派生）🔄 Recall 时重新计算由保留的温度/气压剖面与目标机场标高重新推导，保持大气剖面一致
地面/中间层/对流层顶风环境参数✅ 保存属于教员输入的配置参数
云和能见度环境参数❌ 不保存云和能见度属于与具体训练场景强绑定的瞬时气象状态，跨场景复用价值低；CAE IOS 使用手册的 Save Scenario 显式保存列表也未包含此项
活跃风暴、微下击暴流、预测/非预测风切变、结冰条件环境参数（瞬时态）❌ 不保存属于触发式瞬时模拟状态，非配置参数；重新加载后应由教员按需重新配置
飞机位置、重量、重心飞机物理状态❌ 不保存Weather Scenario Save/Recall 仅保存环境参数，飞机状态由课程、FMS Memory 或教员单独管理
状态持久化契约：Save Scenario 操作的消息序列为：Instructor → CB → SSM → 持久化存储。SSM 在保存前需过滤掉"瞬时模拟状态"，只持久化"教员输入的配置参数"。Recall 时，SSM 从持久化存储读取配置参数，按 Weather Scenario 保存范围对应的初始化顺序（参考机场 → 环境参数 → 物理校验）重新下发至 SC，其顺序与 RF-03 中"机场与环境初始化"阶段保持一致；校验通过后递增 State Version，通知 Control Board、Map 等所有工作区刷新。若校验失败，SSM 保持当前合法状态不变并向教员返回错误提示。
