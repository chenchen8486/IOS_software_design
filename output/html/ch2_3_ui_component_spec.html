基础控件族（Basic Controls）

控件名英文类型状态定义与交互规则
动作按钮 Action Button 瞬时触发 正常态（灰底）→ 点击瞬时触发 → 自动回弹至正常态。无持久状态。典型：Execute、End Lesson、Send Uplink、Capture Flight
切换按钮 Toggle Button 双稳态切换 正常态（灰底）↔ 激活态（浅蓝底/高亮）。点击即切换，无需额外确认。典型：Flight Freeze、Fuel Freeze、Chocks、Jacked、External Pneumatic、Stop Bar、Marshaller
故障切换 Toggle Fail 双稳态切换（故障语义） 与 Toggle 按钮形态相同，但语义为"正常/故障"两种状态。典型：VHF Ground Fault、HF Ground Fault、SATCOM/GES Fault
扩展按钮 Expandable Button 触发弹出容器 正常态 → 点击展开 Sliding Panel → 按钮保持高亮或不变（视具体实现）。典型：控制面板右侧 Expand Button、各 Tab 中展开滑动面板的入口按钮
快速访问按钮 Quick Access Button 快捷导航/触发 与 Action Button 类似，但位置固定在 Quick Access Bar 或 Advanced Toolbar 中，承担高频捷径功能。典型：DL/UL、VHF、Weight&CG、Malfunctions、Failures
参数输入文本字段 Parameter Input Text Field 数值/文本输入入口 只读显示态（当前值）→ 点击 → 弹出 Keypad / Wheel Picker / Sliding Panel。字段本身不直接编辑，仅作为入口。典型：ASL、IAS、GW、Airport ICAO、Surface Wind
只读字段 Read Only Field 纯信息展示 仅显示计算结果或系统状态，不可点击编辑。典型：GW CG（自动计算）、Performance Data 中的 V1/VR/V2
滑块 Slider 连续值调节 最小值端 → 拖拽 → 最大值端，支持实时预览。典型：Position Slew Rate Slider（1~5 档）、Keypad 内的数值滑块、Intruder Number Slider

选择类控件族（Selection Controls）

控件名英文类型状态定义与交互规则
轮选器 Wheel Picker 离散值滚动选择 隐藏态 → 长按/上下滑动参数字段 → 呼出态（滚轮浮层）→ 滚动选择 → 松手即生效（部分参数）或需确认。典型：Time（黎明/白天/黄昏/夜间）、Airport、Runway、Visibility、Temperature、Precipitation
单选按钮 Radio Button 多选一互斥 同一组内仅允许一个激活态（蓝底白点），点击另一项自动取消前项。典型：Weather Presets（CAVOK/CAT I/II/III）、ICING 等级（轻度/中度/重度）、TURBULENCE 类型（CLEAR AIR/THUNDERSTORM/LOW LEVEL）
复选框 Checkbox 多选组合 独立开关项，可多选。典型：Auto Match Reference Airport（INSTRUCTOR STATION Sliding Panel）、地图覆盖层选项
下拉按钮 Dropdown Button 展开列表选择 收起态 → 点击展开下拉列表 → 选择目标项 → 列表自动收起并生效。典型：Preferences 中的力单位/重量单位/环境单位、Taxi Lights（0~5）、City Lights（0~5）
卷展菜单 Roll-up Menu 底部向上展开菜单 与 Dropdown 形态相似，但展开方向为从下向上（常见于 Footer 区域）。典型：Help Roll-up Menu（11.13）、Workspace Switcher Menu
键盘 Keypad 精确数值输入面板 关闭态 → 点击参数字段 → 打开态 → 数字按键输入 / 滑块调节 → 绿勾确认 / 红叉取消 → 关闭。典型：GW Keypad、Wind Keypad、Temp Keypad、QNH Keypad、Flight Level Keypad
交互式图表 Interactive Graph 图形化参数编辑 在坐标系内拖拽节点或曲线来修改参数，实时反馈数值变化。典型：Weight & CG 包线图、UPRT 的 Alpha-Beta 图、V-n 图

容器类控件族（Container Controls）

控件名英文类型状态定义与交互规则
选项卡 Tab 页面内视图切换容器 当前态（高亮下划线/底色）与 非当前态（灰底）。点击切换内容区域，不触发全局状态变更。典型：Control Board 的 5 个 Tabs（LESSON PLAN / DOCS / CURRENT CONDITIONS / AIRCRAFT / REFERENCE AIRPORT）、Sliding Panel 内的子 Tabs（LOG / NEW UPLINK / CONNECTIVITY CONTROL）
滑动面板 Sliding Panel 侧边弹出详情层 收起态（完全隐藏，仅 Expand Button 可见）→ 点击 Expand Button → 展开态（从右侧滑入，覆盖主内容区）→ 内部操作 → 点击关闭按钮/点击外部区域 → 收起态。典型：Malfunctions、Weight & Balance、Airport & Reposition、Clouds、ICING、TURBULENCE、INSTRUCTOR STATION
快速访问栏 Quick Access Bar 底部常驻快捷栏 始终可见，承载高频操作按钮。典型：Control Board Footer（冻结/重置/模拟控制/音量/工作区切换）、Advanced Toolbar（飞机/位置/环境/入侵者/通信/故障/UPRT/重置/录制）
控件计数统计：截至 ch1 全部 HTML 完成时，IOS 教员台界面中共出现 14 种可复用控件类型，其中基础控件族 4 种、选择类控件族 7 种、容器类控件族 3 种。Sliding Panel 作为容器控件，在 10+ 个不同功能模块中被复用，是 IOS 中最高频的容器模式。

2.3.2 控件状态机统一规范

所有 IOS 控件在交互过程中均遵循一个通用的四态流转模式。本节先定义通用模板，再为三种最复杂的控件（Sliding Panel、Keypad、Wheel Picker）绘制独立状态机。

通用控件状态机模板

stateDiagram-v2 [*] --> Idle: 页面加载完成 Idle --> Active: 用户点击/触摸 Active --> Modified: 值已修改未确认 Modified --> Committed: 点击确认或失去焦点 Modified --> Idle: 点击取消或ESC Committed --> Idle: 生效完成 Active --> Disabled: 控件被禁用 Disabled --> Idle: 禁用解除 Committed --> FrozenBuffered: 冻结期间提交 FrozenBuffered --> Committed: 解冻后批量生效 FrozenBuffered --> Idle: 取消预调整

状态说明

状态说明典型控件表现
Idle 空闲态控件未被交互，显示当前生效值Text Field 显示当前参数值；Toggle 显示正常/激活状态；Sliding Panel 完全收起
Active 激活态用户已点击/触摸控件，进入可编辑状态Text Field 高亮边框；Keypad 弹出；Wheel Picker 呼出；Radio Button 高亮候选项
Modified 已修改态用户已输入/选择新值，但尚未确认Keypad 中数字已输入但绿勾未点击；Slider 已拖拽但松手未确认；参数显示"待生效"提示
Committed 已确认态用户点击确认，值已提交至状态管理层Keypad 关闭，主界面字段更新为新值；Toggle 切换完成
Disabled 禁用态控件在当前条件下不可用按钮灰色不可点击；字段置灰；飞机移动时 Chocks/Jacked 禁用
FrozenBuffered 冻结缓冲态冻结期间用户预调整了参数，但变更未下发字段显示预调整值与当前值的差异高亮；解冻瞬间批量生效

2.3.5 设计原则 → 控件设计决策映射

以下表格将 1.1.4 ~ 1.1.11 中总结的五大设计原则，具体化为 2.3 控件体系规范的约束条件。每一条原则都直接影响了控件类型的选择、布局和行为定义。

设计原则原则描述对应的控件设计决策
原则一：按认知模态分离
(Cognitive Modality Separation)		 离散参数与连续空间信息绝不混在同一面板，因为它们激活人脑不同的认知区域 2.3.3 映射表：离散枚举参数（天气预设、时间段）强制使用 Wheel Picker / Radio Button；连续数值参数（亮度、速率）强制使用 Slider；精确数值（重量、温度）强制使用 Keypad。禁止在单个输入流程中混合使用认知模态冲突的控件（如不允许在 Keypad 中同时拖拽地图）
原则二：按使用频率分层
(Frequency-Based Layering)		 高频功能常驻主界面；低频功能独立隔离，减少认知噪声 2.3.1 容器控件：高频操作（冻结/重置/工作区切换）使用 Footer 的 Toggle / Roll-up Menu，常驻可见；中频操作（修改飞机/环境参数）使用 Sliding Panel，点击即达但不常驻；低频操作（UOM 设置、配置切换）使用 INSTRUCTOR STATION Sliding Panel，需要两步才能触及
原则三：按角色隔离
(Role-Based Isolation)		 不同用户角色的功能绝不混用，通过独立工作区 + 权限控制实现干净的安全边界 2.3.4 复用映射：Maintenance 工作区的控件体系与核心训练域完全隔离，教员界面的控件规范不覆盖维护界面。Control Board 中的 Toggle / Keypad 等控件在 Maintenance 模式下不可见，避免角色越界
原则四：同一模型，多重视图
(Multiple Views of One Model)		 同一个领域对象值得为其设计多个正交视图，不强迫用户在"看细节"和"看全局"之间做取舍 2.3.1 容器控件：课程状态同时提供 LESSON PLAN Tab（列表视图，Step-by-Step 控制）和 Lesson Plan Profile（剖面视图，全局进度概览），两者共享同一套底层数据模型，但使用不同的容器控件（Tab vs 独立工作区）。Weight & CG 同时提供 Keypad（精确数值输入）和 Interactive Graph（图形化包线拖拽），满足不同认知偏好
原则五：应急通道设计
(Emergency Bypass Design)		 常规层级导航在紧急场景下是致命的，必须为关键操作预留"直达电梯" 2.3.1 基础控件：Flight Freeze / Total Reset / Clear All Malf. 等最高频应急操作使用 Footer 的 Toggle / Action Button 常驻显示，无需进入任何 Sliding Panel 或 Tab。Advanced Toolbar 的 Quick Access Button 提供 Control Board 功能的直达入口（如 Failure、Environment），交互路径从 3 步缩短至 1 步
对后续章节的输入：2.3 节建立的控件库清单、状态机模板、参数输入范式和复用映射，是 2.4（数据流与接口契约推导） 的 UI 层输入——接口设计中的请求字段、事件类型和状态码应与控件状态机的各态（Idle/Active/Modified/Committed/Disabled/FrozenBuffered）一一对应。
