## 为什么需要本地图片？

A320.md 中的图片链接来自飞书（Feishu）内部 API（`internal-api-drive-stream.feishu.cn`），这些链接带有临时认证码（authcode），**仅限登录用户访问**，外部浏览器和命令行工具均无法直接下载或显示。

因此，HTML 文件已统一改为引用**本地相对路径**。您需要手动将原始图片保存到本目录，HTML 即可正常显示。

## 操作步骤

### 步骤 1：打开 A320.md 原始文档

使用支持 Markdown 预览的编辑器（如 VS Code、Typora），或直接通过飞书文档访问 A320.md 的原始来源。

### 步骤 2：定位对应图片

根据上表中的「对应 A320.md 位置」，找到对应行的图片。

### 步骤 3：右键保存图片

在 Markdown 预览或飞书文档中，**右键点击图片 → 另存为**，保存到以下路径：

```
D:\project\python_release\avation_doc\assets\images\
```

**文件名必须严格按照上表中的「保存文件名」命名**，否则 HTML 无法找到图片。

### 步骤 4：验证

保存完成后，用浏览器打开对应的 HTML 文件：

```
output\html\ch2_2_1_lesson_plan_tab.html
```

确认图片已正常显示。

## 命名规范

为保持工程整洁，所有图片统一采用以下命名格式：

```
ch{章节号}_{子章节号}_{描述}.png
```

例如：
- `ch2_2_1_lesson_plan_overview.png` — 第 2.2.1 章的课程计划总览图
- `ch2_2_3_current_conditions_clouds.png` — 第 2.2.3 章的云层滑动面板图

---

## 注意事项

1. **格式建议**：原始图片保存为 `.png` 格式（保留透明度，适合 UI 开发参考）。如果原始是 `.jpg`，也可使用 `.jpg`，但需同步修改 HTML 中的文件扩展名。
2. **分辨率**：尽量保存原始分辨率，不要压缩，以便 UI 设计师查看细节。
3. **后续章节**：每新增一个 HTML 章节，都会在本文件中追加对应的图片映射表。
